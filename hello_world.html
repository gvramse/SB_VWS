Prerequisites

Before you begin: Make sure you have the following tools installed and configured.
1
Kubernetes Cluster

You need a Kubernetes cluster running. Options include:

  • Minikube: Local development cluster
  • Docker Desktop: Built-in Kubernetes
  • Cloud providers: GKE, EKS, AKS
  • Kind: Local cluster using Docker
2
kubectl CLI

Install the Kubernetes command-line tool:

# macOS (using Homebrew)
brew install kubectl

# Linux
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl
sudo mv kubectl /usr/local/bin/
3
Docker

Install Docker for building container images:

# macOS
brew install --cask docker

# Linux
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
4
Verify Installation

Test your setup:

kubectl version --client
kubectl cluster-info
docker --version

Hello World Application

What we'll build: A simple web application that displays "Hello World" and basic system information.
1

Create the Application

First, let's create a simple Node.js application:

app.js
const express = require('express');
const os = require('os');
const app = express();
const port = process.env.PORT || 3000;

app.get('/', (req, res) => {
  res.send(`
    <h1>üöÄ Hello World from Kubernetes!</h1>
    <p>Hostname: ${os.hostname()}</p>
    <p>Platform: ${os.platform()}</p>
    <p>Architecture: ${os.arch()}</p>
    <p>Uptime: ${Math.floor(os.uptime() / 60)} minutes</p>
    <p>Memory: ${Math.round(os.freemem() / 1024 / 1024)} MB free</p>
    <p>Timestamp: ${new Date().toISOString()}</p>
  `);
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
package.json
{
  "name": "kubernetes-hello-world",
  "version": "1.0.0",
  "main": "app.js",
  "scripts": {
    "start": "node app.js"
  },
  "dependencies": {
    "express": "^4.18.2"
  }
}
2

Create Dockerfile

Create a Dockerfile to containerize the application:

Dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3000

CMD ["npm", "start"]
# Build the Docker image
docker build -t kubernetes-hello-world:latest .

# Test locally
docker run -p 3000:3000 kubernetes-hello-world:latest

Deployment

3

Create Kubernetes Deployment

Deploy the application to Kubernetes:

deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: hello-world-deployment
  labels:
    app: hello-world
spec:
  replicas: 3
  selector:
    matchLabels:
      app: hello-world
  template:
    metadata:
      labels:
        app: hello-world
    spec:
      containers:
      - name: hello-world
        image: kubernetes-hello-world:latest
        ports:
        - containerPort: 3000
        env:
        - name: NODE_ENV
          value: "production"
        resources:
          requests:
            memory: "64Mi"
            cpu: "250m"
          limits:
            memory: "128Mi"
            cpu: "500m"
# Apply the deployment
kubectl apply -f deployment.yaml

# Check deployment status
kubectl get deployments
kubectl get pods

# View logs
kubectl logs -l app=hello-world

Service

4

Create Kubernetes Service

Expose the application internally:

service.yaml
apiVersion: v1
kind: Service
metadata:
  name: hello-world-service
  labels:
    app: hello-world
spec:
  type: ClusterIP
  ports:
  - port: 80
    targetPort: 3000
    protocol: TCP
  selector:
    app: hello-world
# Apply the service
kubectl apply -f service.yaml

# Check service status
kubectl get services

# Port forward to access locally
kubectl port-forward service/hello-world-service 8080:80
Access your application: Open http://localhost:8080 in your browser to see the Hello World application running in Kubernetes!

Ingress

5

Create Ingress (Optional)

Expose the application externally:

ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: hello-world-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  rules:
  - host: hello-world.local
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: hello-world-service
            port:
              number: 80
# Apply ingress (requires ingress controller)
kubectl apply -f ingress.yaml

# Add to /etc/hosts for local testing
echo "127.0.0.1 hello-world.local" | sudo tee -a /etc/hosts

Monitoring

6

Monitor Your Application

Useful commands to monitor your deployment:

# Check deployment status
kubectl get deployments
kubectl describe deployment hello-world-deployment

# Check pods
kubectl get pods -l app=hello-world
kubectl describe pod <pod-name>

# Check services
kubectl get services
kubectl describe service hello-world-service

# View logs
kubectl logs -l app=hello-world
kubectl logs -f -l app=hello-world

# Check resource usage
kubectl top pods
kubectl top nodes

Troubleshooting

Common Issues and Solutions:
Pod Issues
# Check pod status
kubectl get pods

# Describe pod for details
kubectl describe pod <pod-name>

# Check pod logs
kubectl logs <pod-name>
Network Issues
# Check service endpoints
kubectl get endpoints

# Test service connectivity
kubectl run test --image=busybox -it --rm --restart=Never -- wget -O- hello-world-service:80

# Check DNS resolution
kubectl run test --image=busybox -it --rm --restart=Never -- nslookup hello-world-service
